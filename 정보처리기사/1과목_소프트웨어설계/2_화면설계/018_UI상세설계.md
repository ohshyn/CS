# 1 요구사항 확인

## 018 UI 상세 설계

### [1] UI 시나리오 문서 개요

UI 상세 설계: UI 설계서를 기반한 모든 화면의 자세한 설계를 진행하는 단계, 반드시 시나리오를 작성

```
# UI 시나리오 문서
	UI 기능 구조, 대표 화면, 화면 간 인터랙션 흐름, 다양한 상황에서의 예외 처리 등을 문서로 정리한 것
	사용자 목표를 달성하기 위한 방법이 순차적으로 묘사
	UI 설계자/인터랙션 디자이너가 작성, 이후 그래픽 디자이너가 시나리오를 바탕으로 디자인, 개발자가 UI 구현
```

### [2] UI 시나리오 문서 작성 원칙

```
# 계층(Tree) 구조/플로차트(Flow Chart) 표기법 작성
	개발자가 전체 UI 기능과 작동 방식을 한 눈에 이해할 수 있도록 구체적으로 작성

# 일반 규칙 정의
	모든 기능에 공통으로 적용될 UI 요소와 인터랙션

# 대표화면 정의
	레이아웃과 그 화면에 속할 기능

# 인터랙션 흐름 정의
	화면 간 인터랙션 순서(Sequence), 분기(Branch), 조건(Condition), 루프(Loop) 등을 명시

# 다양한 테스트 케이스 정의
	예외 상황에 대비

# 상세 시나리오 정의
	UI 일반 규칙을 지키며 기능별로 정의

# UI 시나리오 규칙 지정
```

### [3] UI 시나리오 문서 작성을 위한 일반 규칙

```
# 주요 키의 위치와 기능
	주요 키: 모든 화면에 공통으로 배치되는 키
	여러 화면 간 일관성 보장

# 공통 UI 요소
	UI 요소(체크 박스, 라디오 버튼, 텍스트 박스 등)*의 사용 형태 정의, 사용자 조작 시 반응 흐름 정의

# 기본 스크린 레이아웃(Basic Screen Layouts)
	모든 화면에 공통으로 나타나는 레이아웃의 위치와 속성을 정의

# 기본 인터랙션 규칙(Basic Interaction Rules)
	공통으로 사용되는 조작 방법과 화면 전환 효과 등을 기술

# 공통 단위 태스크 흐름(Task Flows)
	많은 기능에 공통으로 사용되는 태스크(삭제, 검색, 매너 모드 상태 등)의 인터랙션 흐름을 설명

# 케이스 문서
	다양한 상황에서 공통적으로 적용되는 시스템의 동작을 정의한 문서
```

```
* UI 요소
	체크 박스(Check Box)
		여러 개의 선택 상황에서 1개 이상의 값을 선택할 수 있는 버튼
	라디오 버튼(Radio Button)
		여러 항목 중 하나만 선택할 수 있는 버튼
	텍스트 박스(Text Box)
		사용자가 데이터를 입력/수정이 가능한 상자
	콤보 상자(Combo Box)
		이미 지정된 목록 상자에 내용을 표시하여 선택/입력이 가능한 상자
	목록 상자(List Box)
		이미 지정된 목록에서 내용을 선택만 가능한 상자
```

### [4] UI 시나리오 문서의 요건

```
# 완전성(Complete)
	누락없이 최대한 상세하게
	시스템의 기능보다 사용자 태스크의 초점에 맞춰 기술

# 일관성(Consistent)
	서비스 목표, 시스템 및 사용자 요구사항, UI 스타일 등이 모두 일관성을 유지

# 이해성(Understandable)
	누구나 쉽게 이해할 수 있도록 설명
	불분명하거나 추상적인 표현은 지양

# 가독성(Readable)
	표준화된 템플릿을 활용해 문서를 쉽게 읽도록
	문서 인덱스에 대한 규칙/목차를 제공
	공백을 활용해 가독성을 높인다
	하이퍼링크 등을 활용해 문서들이 서로 참조 되도록

# 수정 용이성(Modifiable)
	시나리오 수정/개선이 쉬워야 한다

# 추적 용이성(Traceable)
	변경 사항의 일자, 대상, 원인을 쉽게 추적 가능하게 작성
```
